História da Web

(Voltar)

Os computadores nem sempre foram avançados como são hoje em dia. As ideias de computadores, antigamente, eram mais primitivas e com menos funções do que estamos acostumados e com todas as facilidades da mnodernidade.

Por mais que existisse a ideia de computador antes disso, foi durante a 2a Guerra Mundial que os computadores mais notáveis existiram, como o Harvard Mark I (1944) e o Colossus (1946), com suas funções voltadas para os cálculos.

Durante essa mesma época, o termo "bug" foi popularizado por conta dos computadores, quando esquentavam demais, atrair insetos que acabavam quebrando a máquina, fazendo com que precisassem ser consertadas manualmente.

Anos depois, conforme a tecnologia evoluia, em meados de 1950, os computadores começavam a diminuir de tamanho quando transicionaram das válvulas para os transistores e, em meados de 1960, já chegando em 1970, os computadores passaram a utilizar chips.

A partir da quarta geração, a partir de 1971, marca desde essa época até os dias de hoje, conforme as evoluções acontecem. Já a primeira conexão da internet, seus primeiros passos eram dados durante a Guerra Fria, em meados dos anos 1950, onde Joseph Licklider

introduziu a ideia de conseguir se comunicar por meio de uma rede, fazendo com que, após algumas pesquisas, a Arpanet fosse criada para conseguir se comunicar entre dois pontos, onde foi possível o acontecimento em 1969 entre duas universidades, mas não foi possível

realizar uma conexão estável no mesmo dia. Como a internet, durante esse período, tinha um uso muito científico, no ano de 1989, Tim Bernes-Lee viu a oportunidade para popularizar a internet utilizando uma forma universal de comunicação por hipertexto, surgindo assim a WWW (World Wide Web).

O que são Clients e Servers

(Voltar)

Clients, nada mais são do que os dispositivos que acessam as informações enviadas pelo Servidor, ou seja, o dispositivo que está consumindo aquela informação que está vindo da internet.

Os dispositivos acessam a internet e, a internet em si, faz a conexão com os servidores, possibilitando que sejam recebidas as informações do servidor direto do lado do Client, exibindo alguns dados.

O servidor enviará algumas informações em HTML, CSS, Javascript, Imagens e outros arquivos estáticos como vídeos e aúdios, que são reproduzidos de forma visual mais limpa para o lado do Client.

Conforme o servidor gera e envia os arquivos, e o client acessa, por meio do navegador, que são programas que interpretam os dados do lado do Client, armazena essas informações em cache para que não precise sempre atualizar e carregar o site completamente de novo.

Os Servers são os servidores que tem um comportamento diferente do Client, que ele serve para o armazenamento de dados e os arquivos que são enviados para o lado do Client quando for acessado. Ele pode servir tanto para guardar arquivos como, até mesmo, para serviços de Streaming.

Linguagens de Programação

(Voltar)

Uma linguagem de programação, em base, é um conjunto de instruções escritas de modo formal que é utilizado para criar programas (softwares). As linguagens de programação são utilizadas como meio para resolver problemas.

É sempre por meio da linguagem de programação que os bugs e diversos problemas são solucionados por meio dos códigos e da análise do que está ocorrendo em si, durante o desenvolvimento do programa.

Exemplos de linguagens que são utilizadas do lado do Servidor:

  • PHP
  • Javascript
  • C#
  • Ruby
  • Java

Exemplos de linguagens que são utilizadas no lado do Client:

  • Javascript

Os códigos podem ser utilizados de forma similar tanto para o lado do Server como para o lado do Client.

Vale lembrar que HTML não é uma linguagem de programação, mas sim, um arquivo de texto com delimitadores que são interpretados pelo navegador.

HTML na Prática

(Voltar)

A estrutura básica do HTML consiste em ter a tag principal "html", onde será inserido todo o texto, o cabeçalho, onde ficará a tag "head", o que carregará antes de ser exibido para o usuário e o corpo, marcado com a tag "body" que é onde a parte visível aparecerá para o Client.

As tags, citadas acima, são indicadores que o navegador interpretará de alguma forma, como, por exemplo, criar um novo parágrafo ou deixar um texto em negrito ou itálico. Toda e qualquer alteração de texto, será indicada por uma tag. O texto estará entre duas tags para que a alteração seja realizada.

As tags também podem ter alguns atributos que são propriedades que as tags podem ter, como identificar um elemento em específico para que esse elemento seja alterado, ou como o atributo "style" que podem formatar o texto e trocar cor de um texto ou a sua fonte.

Existem tags muito comuns de serem utilizadas, como "h1" que pode ir até o "h6" que servem para a criação de títulos e subtítulos, bem como sublinhar com a tag "u" ou marcar um texto em amarelo para dar destaque com a tag "mark"
